# Завдання 5: Візуалізація обходів дерева

Це завдання демонструє роботу алгоритмів **DFS (Depth-First Search)** та **BFS (Breadth-First Search)** через динамічне розфарбовування вузлів бінарного дерева в залежності від порядку їх відвідування.

---

## Логіка роботи

### 1. Алгоритми обходу

Згідно з вимогами, ми відмовилися від рекурсії на користь ітеративних методів, що дозволяє чітко контролювати стан структур даних:

* **DFS (Пошук у глибину):** Використовує **Стек (LIFO)**. Ми занурюємося максимально глибоко по одній гілці, перш ніж повернутися назад.
* **BFS (Пошук у ширину):** Використовує **Чергу (FIFO)**. Ми відвідуємо всі вузли на поточному рівні, перш ніж перейти до наступного.

### 2. Візуалізація

Для кожного обходу генерується динамічний градієнт, що візуалізує "час" виявлення вузла:

* **Градієнт:** Перший відвіданий вузол отримує найтемніший відтінок (базовий RGB), а кожен наступний стає світлішим. Останній вузол стає майже білим.
* **Формат:** Колір автоматично конвертується у 16-кову систему (**HEX**), наприклад `#1296F0`, для коректного відображення у бібліотеці `NetworkX`.
* **Анімація:** Використано модуль `matplotlib.animation` для покрокового оновлення стану графа, що дозволяє спостерігати за алгоритмом у реальному часі.

---

## Аналіз складності

* **Часова складність:** $O(V + E)$, де $V$ — кількість вузлів, $E$ — кількість ребер. Кожен вузол та кожне ребро відвідуються рівно один раз.
* **Просторова складність:** $O(V)$ для зберігання черги/стека та списку відвіданих вузлів у найгіршому випадку (наприклад, для виродженого дерева).

## Висновки

Візуалізація чітко показує фундаментальну різницю в порядку відвідування вузлів:

* **DFS** фокусується на дослідженні гілки до самого кінця перед поверненням (вертикальний підхід).
* **BFS** рівномірно заповнює шари дерева, відвідуючи всіх сусідів поточного рівня перед переходом глибше (горизонтальний підхід).

Використання кольорів робить ці абстрактні поняття інтуїтивно зрозумілими.

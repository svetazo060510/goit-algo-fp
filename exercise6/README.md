# Завдання 6: Жадібні алгоритми та динамічне програмування

Це завдання демонструє порівняння двох класичних підходів до оптимізації — **Жадібного вибору** та **Динамічного програмування** — на прикладі задачі вибору страв із максимальною калорійністю при обмеженому бюджеті.

---

## Опис алгоритмів

### 1. Жадібний алгоритм (`greedy_algorithm`)

Алгоритм сортує страви за їхньою "ефективністю" (співвідношення калорій до вартості). Він обирає найкращі варіанти, доки дозволяє бюджет.

* **Складність:** $O(n \log n)$ через необхідність сортування.
* **Колір у консолі:** Жовтий.

### 2. Динамічне програмування (`dynamic_programming`)

Вирішує задачу про "0/1 рюкзак", гарантуючи знаходження абсолютної максимальної калорійності шляхом побудови таблиці оптимальних станів.

* **Складність:** $O(n \cdot \text{budget})$.
* **Колір у консолі:** Маджента (пурпурний).

## Висновки

1. **Точність:** Динамічне програмування завжди забезпечує **глобальний оптимум**, гарантуючи найкращу можливу комбінацію страв.
2. **Швидкість:** Жадібний алгоритм працює значно швидше на великих обсягах даних, проте він може давати менш точний результат, оскільки "не бачить" вигідних комбінацій дешевших страв, що сумарно дають більше калорій.
3. **Ефективність:** Для стандартних касових систем або швидких розрахунків жадібний підхід є прийнятним, але для критично важливих задач оптимізації ресурсів слід обирати динамічне програмування.
